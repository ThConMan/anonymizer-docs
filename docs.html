Documentation

Everything Anonymizer does, in plain English. Built for Paper / Minecraft 26.x, Java 21+.

Overview

Anonymizer makes invisibility actually mean invisibility. On competitive servers, an invisibility potion hides your body but not your name — kill someone and chat, the death message, the tab list, and your floating nametag all announce exactly who you are. Anonymizer replaces a hidden player's name with a flickering redacted block everywhere it would otherwise show, so a stealth kill stays anonymous.

Anonymizer never reveals more than it has to. The default tab mode keeps players listed but strips their skin to a generic head — the real skin is never sent to other clients while a player is hidden.

Install

  1. Download Anonymizer-x.y.z.jar.
  2. Drop it in your server's plugins/ folder (remove any older Anonymizer jar).
  3. Restart the server.
  4. Adjust plugins/Anonymizer/config.yml and run /anonymizer reload. That's it.

Optional soft-depends (all auto-detected by reflection, no hard dependencies): EssentialsX, CMI, VentureChat.

The trigger

A player is anonymized whenever they carry the Invisibility potion effect. That's the only requirement by default.

For servers that want true ghosts only, turn on require-fully-invisible: a player is then anonymized only when they're invisible and wearing no armor and holding nothing — matching vanilla's "fully invisible" rule. Anyone with the anonymizer.bypass permission is never anonymized.

The moment the Invisibility effect ends, the player's real name returns everywhere automatically.

What gets redacted

The redacted name

A hidden player's name becomes a flickering bold-white block using Minecraft's &k obfuscation. Configure redacted-name.flicker, redacted-name.glyph, and redacted-name.length, or disable the flicker for a clean, solid bar.

Chat

The sender's name is redacted in chat. Anonymizer listens at MONITOR priority (last) and replaces by content, so the redaction survives other chat plugins' formatting — including gradient titles from plugins like DragonReign.

Death messages

If a hidden player is the killer or the victim, their real name is scrubbed out of the broadcast death message before it reaches the server.

Tab list

tab-mode: hide-skin (default) keeps the player listed but strips their skin so the head shows a generic default. Other modes: name-only (redact the name only), remove (drop them from the list entirely), and off.

Nametag

The floating in-world username is hidden through a scoreboard team, so a hidden player's name can't be read through walls or across distance.

Plugin name sources

Nicknames from EssentialsX, CMI, and VentureChat are redacted too — see Plugin hooks.

Config

The whole config.yml is commented in plain language. The key knobs:

trigger:
  require-fully-invisible: false   # true = invisible AND no armor AND empty hands

redacted-name:
  flicker: true        # animate the block with Minecraft's &k obfuscation
  glyph: "█"            # the character the block is built from (solid bar)
  length: 6            # how many glyphs wide the redacted name is

chat:
  enabled: true        # redact the sender's name in chat (runs last, MONITOR)

death-messages:
  enabled: true        # scrub a hidden killer/victim's real name

tab:
  tab-mode: hide-skin  # hide-skin | name-only | remove | off

nametag:
  enabled: true        # hide the floating in-world username (scoreboard team)

hooks:
  essentialsx: true    # redact nicknames in /list, /seen, /near, /whois
  cmi: true            # redact CMI nicknames
  venturechat: true    # redact the sender name in VentureChat

Plugin hooks

All hooks are auto-detected through reflection, so Anonymizer has no hard dependencies and runs fine without any of them.

PluginWhat Anonymizer does
EssentialsXRedacts the hidden player's nickname in /list, /seen, /near, and /whois.
CMIRedacts the hidden player's CMI nickname.
VentureChatRedacts the sender's name in VentureChat channels.
ChatControl / DeluxeChat / CarbonChatDetected and logged with a short config tip so you can wire the redaction into their formats.
Some chat plugins build their messages entirely in their own pipeline. For ChatControl, DeluxeChat, and CarbonChat, Anonymizer logs a one-line tip on startup explaining how to slot the redacted name into their format strings.

FAQ

Does it work without any chat plugin?

Yes. The chat, death message, tab, and nametag redactions are all native — no other plugin required.

Can someone still see the real skin in tab?

No. In the default hide-skin mode the real skin is never sent while a player is hidden; the head shows a generic default.

What if another plugin re-adds the name?

Anonymizer's chat listener runs last (MONITOR) and replaces by content, so it overrides formatting added earlier in the chain.

How do I exempt staff?

Grant anonymizer.bypass. Those players are never anonymized, invisible or not.
